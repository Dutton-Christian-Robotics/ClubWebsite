Rover v2 Capabilities

Motor Commands

Wheels

Numbers used are not degrees; roughly, they represent the speed of motor. Once set in motion, the motor will continue to spin until stopped, either by setting speed in the opposite direction or by issuing a stop command. To best program motion you'll need to experiment with how many speed commands to broadcast and how long to wait before sending a stop command.

  • LEFT CCW1
  • LEFT CW1
  • LEFT STOP
  • RIGHT CCW1
  • RIGHT CW1
  • RIGHT STOP
Grabber Arm

Numbers are in degrees. The arm begins at 0, which is fully retracted. At 180° counter-clockwise, the arm is fully extended. The arm animates at several points along the way.

  • ARM CCW1
  • ARM CW1
Grabber Claw

Numbers are in degrees. The claw starts at 0, which is fully open. At 90° clockwise, the claw is fully closed. There is no animation except for open and closed states.

  • CLAW CCW1
  • CLAW CW1

Sensor Commands

Long Range Sensors

The rover is equipped with distance sensors pointing forward, back, left, and right. They can be used to calculate distance toward obstacle in those directions. For the sensors to work, they must be specifically called using broadcast commands.

  • FRONT LRS
  • BACK LRS
  • LEFT LRS
  • RIGHT LRS

Active Sensors

Long Range Sensors

Sensor values are accessed through Scratch variables. All long range sensors use –1 as a default "distance not calculated" value. 0 and positive numbers measure the distance from the bot to any obstacles in that direction. The bot sprite has a sensitivity setting; increasing that value that will make the sensors faster but less accurate.

  • FRONT LRS DISTANCE
  • BACK LRS DISTANCE
  • LEFT LRS DISTANCE
  • RIGHT LRS DISTANCE
Vision Sensors

This variable contains a value of 1 if there is red (a heart) directly in the forward path of the bot. This sensor value is set after calling FRONT LRS. Note that only the front sensor detects color.

  • SEEING RED

Passive Sensors

General Contact Sensors

These are passive sensors, which mean you do not need to issue any command for the values to be available to you. All are boolean values, 0 or 1. This set of sensors is activated by any contact with any part of the bot. Separate sensors allow for detecting what part of the rover was touched.

  • TOUCHING EDGE
    This sensor variable should almost never actually be 1. The backgrounds for this simulation are built with purple obstacle walls around the edge which should prevent "edge" detection.
  • TOUCHING OBSTACLE
    Walls and obstacles are purple. If any part of the bot is touching an obstacle, this sensor variable will be 1.
  • TOUCHING RED
    Similar to obstacle detection, this sensor variable will be 1 if any part of the bot is touching a red heart.
Directional Contact Sensors

These sensor variables correspond to the yellow bars around the bot. They return 1 if that particular sensor is touching EITHER red or a purple obstacle. Note that each directional sensor could be touching both red and purple. The sensors themselves do not let you know which is which.

  • FRONT TOUCHING
  • BACK TOUCHING
  • LEFT TOUCHING
  • RIGHT TOUCHING
Claw Contact Sensors

This sensor variable returns 1 if the tip of the claw (which is also red) is touching a red heart. This sensor will only work when the grabber arm is fully extended.

  • CLAW TOUCHING

Miscellaneous Commands

Quick Setup

Broadcasting this command doesn't technically control the rover. It simply acts as a convenient way to put the rover back in the middle of the stage, facing direction 0.

  • RESET LOCATION
Resetting Sensors

Some sensor values are "sticky" meaning that they keep their positive value until updated. Broadcasting this command will manually reset all the bot's sensors to their default values (–1 for distance sensors, 0 otherwise).

  • RESET SENSORS
