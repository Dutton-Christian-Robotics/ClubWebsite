Rover v2 Reminders

DO
  • Issue commands to the bot by broadcasting messages
  • Receive feedback from sensors by examining variable values
  • Separate code into specific functions using custom blocks
  • Use multiple flag blocks to create code that you want to run at the same time—e.g. code to control movement and code to do sensor checking
DO NOT
  • Do not alter any code for the Bot, Heart, or the LR Sensor sprites
  • There should be absolutely NO Motion blocks in the keyboard sprite code. To move the bot, you need to broadcast commands to it.
  • There should be absolutely NO contact or color sensing blocks in the keyboard sprite code. You need to use the sensor commands and variables to learn about the bot's environment.
REMEMBER
  • Because this is a simulation, long range sensor checking is not instantaneous. You can change the sensitivity in the bot sprite, but that will make it more inaccurate. When sending a sensor "ping" be sure to wait a second or two for the response.
  • Nothing stops the bot from driving over obstacles. It's up to you to prevent that.
